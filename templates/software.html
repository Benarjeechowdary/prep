Software Engineer Interview Questions

  • Explain the difference between object-oriented and functional programming.
    OOP models data as objects with properties and behaviors, while functional programming focuses on using pure functions and immutable data.
  • How would you optimize a slow database query?
    Use indexing, query optimization techniques, proper joins, and caching to improve performance.
  • What are the four pillars of OOP?
    Encapsulation, Abstraction, Inheritance, and Polymorphism.
  • How does garbage collection work in Java/Python?
    Garbage collection automatically frees up memory by removing objects that are no longer referenced.
  • Explain the SOLID principles.
    SOLID stands for Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, and Dependency Inversion.
  • What is a deadlock in multithreading?
    A deadlock occurs when two or more threads are waiting for each other to release resources, causing a standstill.
  • How does HTTP differ from WebSockets?
    HTTP is a request-response protocol, while WebSockets allow for full-duplex communication over a single connection.
  • Explain how hashing works in data structures.
    Hashing converts input data into a fixed-size hash value using a hash function to enable fast data retrieval.
  • What is the difference between an interface and an abstract class?
    An interface defines a contract that implementing classes must follow, while an abstract class can have both abstract and concrete methods.
  • How would you handle memory leaks in an application?
    Use profiling tools, avoid circular references, and release unused objects to prevent memory leaks.
  • Explain the purpose of Docker and Kubernetes.
    Docker allows containerization of applications, and Kubernetes manages container orchestration and scaling.
  • How do you implement authentication and authorization in a web app?
    Use JWT tokens, OAuth, or session-based authentication, and control access with role-based permissions.
  • Describe a time when you debugged a complex issue.
    Example: Tracing a performance issue to a misconfigured database index and resolving it.
  • How do you handle API rate limiting?
    Use a token bucket or leaky bucket algorithm to control request rates and return appropriate status codes.
  • What is the CAP theorem in distributed systems?
    The CAP theorem states that a distributed system can only provide two of three guarantees: Consistency, Availability, and Partition Tolerance.
